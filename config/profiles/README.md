# Role Profiles

This directory contains the 14 role profiles that drive `Idu-pi`'s supervisor and AgentLab roles. Each profile is the contract for one role: who it is, what it reads, what it produces, how it works, what it has prohibited, who wakes it, and what model it uses by default.

The profiles are versioned copies of the upstream source at `C:\Users\elmas\Downloads\Documento proyecto\Perfil ia\skill-*.md`. The upstream is the source of truth; this directory is what the runtime reads.

## Sync protocol

1. The upstream `Perfil ia/` is edited by hand.
2. This directory is regenerated by copying the upstream into the repo (the `rol-id` field of each profile's frontmatter is the filename, so `skill-orquestador.md` becomes `orchestrator.md`, `skill-supervisor-principal.md` becomes `supervisor-main.md`, etc.).
3. `git diff` on this directory shows the contract change. The diff is what the reviewer audits.

The first version of this directory was created 2026-06-15 from the upstream copy. No automation exists yet (`pnpm run sync-profiles` is a deferred follow-up).

## Loader

`src/roles/profile-loader.ts` reads these files. Each profile has a YAML-ish frontmatter (no third-party deps), followed by the markdown body. Sections: **Quién soy / Qué leo / Qué produzco / Cómo trabajo / Qué tengo prohibido / Quién me despierta / Modelo**.

The loader exposes a typed `RoleProfile` to the role modules. The `prompt-helpers.ts` shim in `src/roles/` is the integration point: existing helper functions continue to work, but they now derive their strings from the loaded profile instead of hardcoded prompts.

See `src/roles/profile-loader.ts` and the test in `test/e2e-skills-loaded.test.ts`.
